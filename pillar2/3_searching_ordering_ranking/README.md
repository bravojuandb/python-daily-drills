# 3. Searching, Ordering & Ranking

This chapter develops precise search and ordering logic, including boundary
conditions and deterministic tie-breaking.

## What This Chapter Should Prove

- implement and compare linear and binary search
- find extrema without sorting unnecessarily
- define compound sort keys
- rely on stable sorting intentionally
- rank aggregated values deterministically
- choose full sorting or a heap for top-k selection

## Drill Order

1. Linear search
2. Binary search
3. Manual min and max
4. Compound-key sorting
5. Stable priority ordering
6. Leaderboard ranking
7. Top-k by full sort
8. Top-k with a heap
