# 6. Applied Challenges

This chapter combines the earlier patterns. The prompts specify outcomes but do
not prescribe every intermediate step; choosing the design is part of the drill.

## What This Chapter Should Prove

- separate validation, transformation, aggregation, and formatting
- combine multiple data structures without unnecessary passes
- preserve deterministic output
- process streams without loading all raw input into memory
- handle malformed records explicitly
- state the dominant time and space costs of a complete solution

## Drill Order

1. Mini leaderboard
2. Unique visitor report
3. Word-frequency report
4. Customer transaction summary
5. Record merge with conflict rules
6. Nested-data flattener
7. Streaming monthly CSV totals
8. ETL capstone
