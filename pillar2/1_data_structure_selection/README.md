# 1. Data Structure Selection

This chapter trains you to select a structure from the operations a problem
requires, rather than reaching for a familiar container automatically.

## What This Chapter Should Prove

- distinguish order, uniqueness, mutability, and hashability requirements
- use tuples as immutable records and composite dictionary keys
- use sets for membership and uniqueness
- use dictionaries as direct indexes
- apply stack and queue semantics deliberately
- explain why one representation fits better than another

## Drill Order

1. Ordered event log
2. Immutable coordinate records
3. Unique tags
4. ID lookup index
5. Composite-key totals
6. Stack-based undo history
7. Queue-based task processing
8. Choose a structure from requirements
