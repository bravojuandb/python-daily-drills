# 8. Input/Output Basics

This chapter is where your code starts interacting with the outside world more directly.

Its job is to make you comfortable reading from users, formatting visible output, and working with files in small, controlled ways.
If this section is solid, then scripts, utilities, and small automation tasks become much easier to build.

## What This Chapter Should Prove

After finishing `8_input_output_basics`, you should be able to:

- read user input and convert it into usable values
- print output in a deliberate, readable format
- read from and write to text files safely with `with`
- separate a small input/output task into a clear function
- think about basic validation before trusting outside data

## Drill Standards

Each drill in this chapter should ideally meet this bar:

- one clear function or tiny program is the main solution
- the prompt states the expected output shape or effect
- the solution uses the target I/O operation directly
- the drill includes one small validation or formatting requirement
- the file stays short enough to solve in one sitting

## Drill Summary

- `01_user_input_echo.py`
  Read one line of user input and print it back immediately.

- `02_number_guessing.py`
  Build a tiny guessing loop with numeric input and comparison feedback.

- `03_print_formatting_challenge.py`
  Format visible output precisely with an f-string.

- `04_simple_file_reader.py`
  Read lines from a text file and present them with numbering.

- `05_simple_file_writer.py`
  Write a small text file and verify the written content.

- `06_append_log_entry.py`
  Append one new line to an existing file without overwriting prior content.

- `07_count_file_lines.py`
  Read a file and return a count of meaningful lines.

- `08_copy_nonempty_lines.py`
  Copy text from one file to another while filtering blank lines.
